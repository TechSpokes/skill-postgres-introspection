# Foundations

The shape of this skill is not incidental. It applies three frameworks developed at TechSpokes by Serge Liatko for getting humans, AI models, and deterministic tools to work from the same understanding. The frameworks are proprietary and used privately by the company. This page describes, at a conceptual level, what each one addresses and how it shaped the skill. It does not reproduce the frameworks.

## The Three-Body Problem of cross-intelligence communication

Humans, AI models, and deterministic tools each interpret the same words through a different mechanism, so an instruction that reads clearly to one can silently mislead another. The framework treats communication across these three kinds of mind as a design problem with predictable failure modes rather than a matter of wording.

How it shaped the skill: the goal is stated before any procedure, terms that carry different meanings in different contexts are defined before use, each workflow step names its purpose so an agent can adapt instead of following blindly, and instructions are kept atomic and front-loaded so the critical part survives partial reading.

## The Universal Problem-Solving Framework

Framing a problem, its current state, its goal, the gap between them, and the constraints and obstacles in the way, is half the work, and sound decisions come from surfacing obstacles to a goal rather than applying rules mechanically.

How it shaped the skill: the diagnose-do-not-prescribe stance, the alignment loop that compares the data layer against the repository's goals, treating a finding such as a missing index as an obstacle to weigh rather than a defect to fix, and bringing decisions to the user instead of acting alone.

## Form Architecture

The structure of a document, its ordering, headings, list shape, and emphasis, is an engineering variable that decides whether meaning survives across different readers and tools. It is not a matter of style.

How it shaped the skill: progressive disclosure from must-know to background, real headings instead of decorative markup, flat atomic lists, and generated output kept minimal and greppable so its heading outline serves as a map a human or a tool can read cheaply.
